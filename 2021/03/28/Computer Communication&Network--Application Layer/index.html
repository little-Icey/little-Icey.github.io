1.Principles of network applications

Application architectures 网络应用架构

  • Client-server（包括data centers/cloud computing）
  • Peer-to-peer(P2P)
  • Hybrid of client-server and P2P(混合架构)

Client-server architecture

  1. server
    • always-on host
    • permanent IP address 固定IP地址
    • Data center for scaling 数据中心
  2. clients
    • communicate with server
    • may be intermittently connected 临时连接
    • dynamic IP 动态地址
    • do not communicate directly with each other 彼此间不直接通信

P2P

  • no always-on server
  • arbitrary end systems directly communicate 相互直接通信
  • peers request service from other peers, provide service in return to other peers 人人为我，我为人人
    • self scalability – new peers bring new service capacity, as well as new service demands 自我扩展
  • peers are intermittently connected and change IP addresses 临时连接，动态地址

Hybrid of client-server and P2P

  • Skype
    • 客户端之间直接建立连接（无需通过中心服务器）
  • Instant messaging (QQ)
    • 用户通过中心服务器找到好友的IP地址

Processes communicating 进程通信

  1. within same host, two processes communicate using interprocess communication (defined by OS). 同一主机上的进程使用进程通信机制
  2. processes in different hosts communicate by exchanging messages 不同主机上的进程通过交换消息进行通信
  3. Client process 发起通信的一方
  4. Server process 等待通信的一方

Sockets 套接字

  1. process sends/receives messages to/from its socket 进程通过套接字接收和发送消息
  2. sending process relies on transport infrastructure on other side of door to deliver message to socket at receiving process 发送进程依赖传输基础设施将消息发送到接收进程的socket

Addressing processes 进程寻址

  • 进程必须有标识符（identifier）
  • host device has unique 32-bit IP address
    如何标识进程：IP地址+端口号

App-layer protocol defines

  1. Types of messages exchanged 报文类型
    • e.g., request, response
  2. Message syntax:报文语法
  3. Message semantics 报文语义
  4. Rules for when and how processes send & respond to
    messages 规则

transport service 传输服务

  1. Data loss
  2. Timing
  3. Throughput
  4. security

Internet transport protocols services 传输层协议提供的服务

TCP service:

  • reliable transport between sending and receiving process 可靠数据传输
  • flow control: sender won’t overwhelm receiver 流量控制
  • does not provide: timing, minimum throughput guarantee, security 不提供延时、最小带宽保证及安全性
  • connection-oriented: setup required between client and server processes 需要在客户和服务器之间建立连接

UDP service:

  • unreliable data transfer between sending and receiving process 不可靠数据传输
  • does not provide: reliability, flow control, congestion control, timing, throughput guarantee, security, or connection setup 不提供可靠性、流量控制、拥塞控制、延时、带宽保证、安全性及连接建立
